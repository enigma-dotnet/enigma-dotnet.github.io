using Netiqo.Framework;

var host = new Host("127.0.0.1", 8080);

Componenti principali e API per lo sviluppo

Classe Host

Classe principale per la gestione del web server con supporto HTTPS opzionale.

Host Constructor
public Host(string ip, int port, string certPath = null, string certPassword = null)

Classe Route

Definizione delle route con supporto per handler dinamici o risposte HTML statiche.

Route Constructor
public Route(string path, Request.HttpMethod method, 
    Func<string, Task<string>> handler = null, 
    string htmlResponse = null)

RouteAttribute

Decoratore per definire route direttamente sui metodi delle classi controller.

Route Attribute Example
[Route("/api/test", Request.HttpMethod.GET)]
public string Test(string body) => "<h1>Test</h1>";

HttpMethod Support

Supporto completo per tutti i metodi HTTP standard:

  • GET, POST, PUT, DELETE
  • PATCH, HEAD, TRACE
  • OPTIONS, CONNECT

Classe Device

Utility per ottenere informazioni di rete e configurazioni predefinite.

Device Utilities
string ip = Device.Ipv4();
int defaultPort = Device.DefaultPort; // 8080

Handler Dinamico

Per contenuto che richiede elaborazione server-side e interazione con dati.

Dynamic Handler

{
    var parsed = HttpUtility.ParseQueryString(body);
    return $"<h1>Benvenuto {parsed["username"]}</h1>";
}));

üìÖ Eventi e Logging

Sistema di logging flessibile e personalizzabile

Sistema di logging flessibile che permette di intercettare e gestire tutti gli eventi del server.

Event Logging
host.LogMessage += msg => Console.WriteLine("[LOG] " + msg);

// Logging avanzato con timestamp
host.LogMessage += msg => {
    var timestamp = DateTime.Now.ToString("yyyy-MM-dd HH:mm:ss");
    Console.WriteLine($"[{timestamp}] {msg}");
};
